[HTMX](https://htmx.org/) is a JavaScript library that lets you create interactive websites entirely out of a single backend server that generates HTML snippets. It works by endowing HTML elements with special "hx" attributes that setup triggers for HTTP requests and targets for where to put HTML responses.

Critically, this can mean avoiding a lot of the complexity surrounding passing data back and forth between a frontend client and a backend server, let alone setting up a frontend client. For more background, see my [earlier post](/posts/simple-htbuilder-htmx-fastapi-combo) about using HTMX with Python's [FastAPI](https://fastapi.tiangolo.com/tutorial/first-steps/) and [Htbuilder](https://github.com/tvst/htbuilder).

This post is an overview of how to build a HTMX powered website, whose backend server written in Haskell's [Servant](https://docs.servant.dev/en/latest/index.html) API framework, and whose HTML is generated using [Lucid](https://hackage.haskell.org/package/lucid2#readme) package (see [Footnote 1](#footnote-1) for tutorial info).

In particular, we'll go over a simple project I made of a "Vomit Draft Text Editor" (all code available in this [public repo](https://github.com/ori-livson/haskell-vomit-draft-editor)). Besides the Haskell implementation, the idea was to pick a project with more complex state and flow than a data entry form / admin UI, which is my usual use case for HTMX.

**Note:** if you're interested in other approaches to web development with functional languages I strongly recommend also looking into [elm](https://elm-lang.org/), and [elixir's Pheonix](https://www.phoenixframework.org/), which is apparently very loved.
