[HTMX](https://htmx.org/) is a JavaScript library that lets you create interactive websites entirely out of a single backend server that generates HTML snippets. It essentially works by endowing HTML elements with special "hx" attributes that setup triggers for HTTP requests and targets for where to put HTML responses.

Critically, this can mean avoiding a lot of the complexity surrounding passing data back and forth between a frontend client and a backend server, let alone setting up a frontend client. For more background, see my [earlier post](/posts/simple-htbuilder-htmx-fastapi-combo) about using HTMX with Python's [FastAPI](https://fastapi.tiangolo.com/tutorial/first-steps/) and [Htbuilder](https://github.com/tvst/htbuilder).

This post is an overview of how to build a HTMX powered website with HTML generated by Haskell's [Lucid](https://hackage.haskell.org/package/lucid2#readme) package (see [Footnote 1](#footnote-1) for tutorial info), and served by an API made with Haskell's [Servant](https://docs.servant.dev/en/latest/index.html) framework.

In particular, we'll go over a simple project I made of a "Vomit Draft Text Editor" (all code available in this [public repo](https://github.com/ori-livson/haskell-vomit-draft-editor)). Besides the Haskell implementation, the idea was to pick a project with more complex client state than a data entry form / admin UI, which is my usual use case for HTMX.

**Note:** if you're interested in other approaches to web development using functional languages, I strongly recommend looking into [elm](https://elm-lang.org/), and also [elixir's Pheonix](https://www.phoenixframework.org/), which is apparently very loved.
